# Documentation

## Scope

Documentation owns understanding guides, README material, ADR maintenance, architecture descriptions, and shared project status.

## Boundaries

- Derive claims from code, tests, Git history, or accepted decisions.
- Do not create an alternate source of project status outside `.ai/`.
- Do not change production code or claim validation that was not run.

## Deliverable

List updated documents, evidence sources, corrected assumptions, cross-links, and any freshness risk that needs an owner.
