# Gemini Repository Guide

All current project knowledge and operating constraints live in `.ai/`; do not create a Gemini-specific version.

Follow this reading sequence:

1. [.ai/PROJECT_STATUS.md](.ai/PROJECT_STATUS.md)
2. [.ai/ARCHITECTURE.md](.ai/ARCHITECTURE.md)
3. [.ai/DECISIONS.md](.ai/DECISIONS.md)
4. [.ai/AGENT_RULES.md](.ai/AGENT_RULES.md)
5. [.ai/TEST_MATRIX.md](.ai/TEST_MATRIX.md)

Afterward, use `.ai/tasks/active/` for the assigned work and `.ai/agents/` for role boundaries. Keep repository facts out of this entry file.
